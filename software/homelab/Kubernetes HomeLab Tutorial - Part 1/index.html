    Kubernetes HomeLab Tutorial - Part 1 - Installing Kubernetes and Addons

    Introduction

    This is a set of tutorials on setting up a Kubernetes cluster at home. At the end, we'll get the following.

    • Kubernetes cluster
    • CI/CD with Github Actions
    • Kafka
    • Logging using Elastic Search, FluentD, and Kibana
    • Metrics and Alerts using Prometheus and Grafana

    I'm working on a Dominoes game as a learning experience. The game consists of multiple applications running on Kubernetes and communicating through Kafka. I wanted to automate the CI/CD process and add common tools such as logging and metrics with minimal configuration to each component.

    I'll keep this tutorial updated with my latest findings.

    Assumptions

    For people following along with this tutorial, there's some assumption I've made.

    • You know the basic of Kubernetes and you're comfortable with kubectl
    • You have a spare computer and you're not running an ARM device (sorry Raspberry Pi). Some of the components we'll add will not work on ARM devices.

    Not For Production

    The cluster is enough for development, but I wouldn't run my production on it for the following reasons.

    Single Node

    The Kubernetes is running on a single system. Though it's easy to add additional nodes with MicroK8s, I'm using a very simple way to allocate storage, and you may need something more complex for a cluster.

    External traffic using Node Ports

    Node Port is the simplest way to get external traffic flowing into the Kubernetes cluster. But there's plenty of downsides to it. This article gives a good overview between Node Port, Load Balancer, and Ingress. Rather that deal with the extra complexity of Ingress, I'll use Node Ports.

    No Authorization

    The cluster isn't using RBAC Authorization. It's hosted at home with the only open port used for kubectl. Adding RBAC adds additional complexity.

    Step 1: Installing the OS - Ubuntu

    The first step is installing the OS. I choose Ubuntu because it makes setting up Microk8s a snap. The Desktop or Server edition are very similar so choose whichever you're comfortable with.

    Change Host Name

    I like to give the computer a descriptive hostname (e.g. k8s-staging-server). Run the following command to update the hostname.

    hostnamectl set-hostname [new-host-name]

    Run hostnamectl to verify it worked.

    SSH Setup

    I usually SSH into the Kubernetes server from my development laptop. Therefore, I needed to add openssh-server. A nifty option is the ability to import your keys from either launchpad or github.

    Step 2: Installing Kubernetes - Microk8s

    Microk8s is a small, fully-conformant Kubernetes that's easy to setup. Installing only requires running the following command.

    sudo snap install microk8s --classic

    It'll take a few minutes to install the application. Run microk8s status to confirm everything is up and running.

    Step 3: Setting up your kubectl

    Run the following command on the Ubuntu system to get your configuration.

    microk8s config

    Copy that information into your development system ~/.kube/config file in whatever method you're most comfortable with. Then try to connect to the Kubernetes cluster from your development system.

    Certificate error

    Most likely you'll get a certificate is invalid error. This post describes some possible solutions. The following steps work for me.

    Edit the certificate template file nano /var/snap/microk8s/current/certs/csr.conf.template and under alt_names add your ips.

    [ alt_names ]
DNS.1 = kubernetes
DNS.2 = kubernetes.default
DNS.3 = kubernetes.default.svc
DNS.4 = kubernetes.default.svc.cluster
DNS.5 = kubernetes.default.svc.cluster.local
IP.1 = 127.0.0.1
IP.2 = 10.151.233.1
IP.100 = <K8S_INTERNAL_IP>
IP.101 = <K8S_EXTERNAL_IP>

    I needed to add both the internal IP (for local access) and external IP (for GitHub CI/CD). The IP IDs must be over 99. Once you save the file, the template is automatically reloaded and you should now be able to connect to the cluster.

    Kubernetes UI

    I highly recommend Visual Studio - Kubernetes addon to view your Kubernetes cluster. But don't worry, you'll still need to keep kubectl nearby.

    Step 4: Addons

    Next, let's add some addons.

    • dns
    • dashboard
    • helm3
    • storage

    The documentation describes the purpose of these addons and how to add them.

    Kubernetes Dashboard

    By default, the Kubernetes Dashboard is not accessible from outside the cluster. The simplest way to change that is to create a NodePort for the service

    Running kubectl get service --namespace kube-system shows the following information.

    NAME                        TYPE        CLUSTER-IP       EXTERNAL-IP   PORT(S)                  AGE
kubernetes-dashboard        ClusterIP   10.152.183.197   <none>        443/TCP                  82s
...

    Run the following command to update the service.

    kubectl edit svc/kubernetes-dashboard --namespace kube-system

    Then update the port from ClusterIP to NodePort and add the port you want to use. Here's an example.

    ...
ports:
  - port: 443
    protocol: TCP
    targetPort: 8443
    nodePort: 30443 # Pick a port number
  selector:
    k8s-app: kubernetes-dashboard
  sessionAffinity: None
  type: NodePort # Changed from ClusterIP
...

    Save the file. The service should be updated. Running kubectl get service --namespace kube-system again should show the following.

    NAME                        TYPE        CLUSTER-IP       EXTERNAL-IP   PORT(S)                  AGE
kubernetes-dashboard        NodePort    10.152.183.197   <none>        443:30443/TCP            17m
...

    Let's access the dashboard by going to your web browser and entering https://[k8s-ip-address]:30443/. You'll see a Warning page because the SSL certificate is self-signed. Continue on and you'll reach the dashboard. There's two ways to authenticate. If you updated your kubeconfig file then choose that authentication option.
