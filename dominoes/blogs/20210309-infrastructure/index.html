    Infrastructure

    Introduction

    The primary infrastructure driver is using Kubernetes to deploy, scale, and manage both the Dominoes system and the development and maintenance of it. Some examples of applications or services deployed to Kubernetes are

    • Database
    • CI/CD runners
    • Logs Collection
    • APM

    I would like to keep the cost as low as possible.

    Home Cluster

    I bought an 8 GB Raspberry Pi 4 for the specific purpose of running a K8s cluster back in mid-2020. Instead of the full Kubernetes, I had it run K3s. I enjoyed the process, but there were a few issues with it.

    ARM64

    The Raspberry Pi 4 uses an arm64 chip. And some popular apps, containers, and Helm charts do not support ARM. Some examples are

    • SonarQube
    • GitLab Runner

    I would love to run my K8s cluster on a few local Raspberry Pis, but I can easily see myself bogged down for weeks on setting it up and getting it to run various applications and services.

    Cloud

    All the major cloud providers has some sort of managed Kubernetes cluster. I'm most interested in Digital Ocean. It's cheap and integrates with GitLab's core components, another piece I was interested in working with.

    Conclusion

    I'm leaning toward starting with Digital Ocean and determining if it fits my need and budget. You'll see plenty of more blog posts on this topic. Maybe I'll build a retro gaming machine with my Raspberry Pi....
