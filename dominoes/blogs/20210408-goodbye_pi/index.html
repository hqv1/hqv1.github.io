    Goodbye Pi

    Raspberry Pi Issues

    I wish I could use my Raspberry Pi 4. But there were two major issues I faced in the week I using it as my primary Kubernetes cluster.

    ARM64 support

    A lot of applications don't support the ARM64 architecture. For example, there aren't helm charts for Kafka. And Confluent Platform doesn't support ARM64 at all. Support should increase now that the new MacBooks are using ARM64 chips, but it doesn't help me now.

    Performance

    The performance is slooooow. Calling a simple Rest API on the Kubernetes cluster takes a minute to warm up and a dozen seconds afterward. And that's only with one pod running in the cluster and Kafka installed on the OS itself. I still had a database and a few other applications I wanted to add onto it.

    Hello Old Computer

    Fortunately, I had an old gaming computer laying around. I installed Ubuntu Desktop on it (Server didn't boot up for me) and I'm a pro at installing MicroK8s now. I'm hoping it can handle the workload, but I can always add in additional computers to the cluster. And save on my heating bill!

    Kafka on Kubernetes

    It was also easy to install Kafka using Helm chart. I wanted something simple to get up and running. I'll probably move to the Confluent Platform at some point to get Schema Registry.

    By default, the Kafka cluster is only reachable within the Kubernetes cluster. There's a few ways to provide external access, but I found this to be the easiest way if you only have one Kafka broker.

    helm upgrade --set externalAccess.enabled=true --set externalAccess.service.type=NodePort --set externalAccess.service.nodePorts[0]='{port}' --set externalAccess.service.domain={internal-ip-address} my-kafka bitnami/kafka

    I needed to add externalAccess.service.domain and use the internal ip address of the Kubernetes cluster. Otherwise the broker will try to access the external listener by the public IP address and I didn't open it up.

    To get a Kafka client pod up and running, execute the following commands.

    kubectl run my-kafka-client --restart='Never' --image docker.io/bitnami/kafka:2.7.0-debian-10-r109 --namespace {namespace} --command -- sleep infinity

kubectl exec --tty -i my-kafka-client --namespace {namespace} -- bash

    Back to the Raspberry Pi

    All is not lost for the Raspberry Pi though. I can still use it as a database server or for some other dominoes related task. Please don't be a paper weight :(
